# config.json schema

The `config.json` file at the repo root drives the entire pipeline. Read it in Phase 1 before doing anything else.

## Full example

```json
{
  "repos": [
    {
      "name": "org/repo-name",
      "github_url": "https://github.com/org/repo-name"
    }
  ],
  "preferred_diagrams": ["component", "sequence", "class"],
  "doc_format": "markdown",
  "analysis_options": {
    "critical_files_limit": 50,
    "include_metrics": true,
    "max_tokens": 5000,
    "timeout_minutes": 10,
    "analysis_profile": "standard"
  },
  "confluence": {
    "enabled": true,
    "site_alias": "your-site-alias",
    "space_key": "YOUR_SPACE",
    "parent_page_title": "Your Parent Page Title",
    "page_suffix": ""
  }
}
```

## Fields

### `repos[]` ŌĆö array of repositories to analyze

- `name`: identifier in `org/repo-name` form. Used as the directory name (with `/` ŌåÆ `_`) under `output/` and as the prefix of the Confluence page title.
- `github_url`: HTTPS URL to the repo. Must be HTTPS (`https://github.com/...`) ŌĆö not SSH (`git@github.com:...`). Phase 1 converts SSH to HTTPS automatically; see `references/url-conversion.md`.

### `preferred_diagrams` ŌĆö array of diagram types to generate

Typical value: `["component", "sequence", "class"]`. Each diagram is generated by `mcp__mermaid_image_generator__*` and saved as `<type>_diagram.png` in the run's output directory.

### `doc_format` ŌĆö output format

Currently always `"markdown"`. The skill produces a `docs.md`.

### `analysis_options`

- `critical_files_limit`: how many top files to surface in the analysis. Typical: 50ŌĆō100.
- `include_metrics`: whether to embed quantitative measures in the docs.
- `max_tokens`: per-section token budget for generated documentation.
- `timeout_minutes`: per-repo soft limit on analysis. The hard timeout is the sub-agent process timeout (5 or 10 minutes).
- `analysis_profile`: one of `standard`, `developer_onboarding`, `architecture_review`, `business_understanding`, `operations_handover`. Determines audience, emphasis, depth, and the suggested page suffix. See `references/profiles.md`.

### `confluence`

Skip this entire block if `enabled: false`.

- `enabled`: master switch. When `false`, local docs+diagrams are still produced but no Confluence upload happens.
- `site_alias`: the alias defined in the user's Conduit config (`~/.config/conduit/config.yaml`). Must match exactly.
- `space_key`: Confluence space key (e.g., `ENG`, `DEMO`).
- `parent_page_title`: the **exact** title of the existing page under which new analysis pages will be created. The pipeline does **not** create the parent ŌĆö if it doesn't exist, Confluence upload fails. Verify before running.
- `page_suffix`: optional string appended to the repository name to form the final Confluence page title. Typically tied to the analysis profile (e.g., ` - Architecture Review`). See `references/confluence-upload.md` for the full title formula.

## Pre-flight checks Phase 1 must do on this file

1. Every `github_url` is HTTPS (not SSH). Convert SSH if needed and inform the user.
2. `analysis_profile` matches one of the five known values.
3. If `confluence.enabled: true`, all four required Confluence fields are present and non-empty.
4. The `analysis_profile` and `page_suffix` are consistent ŌĆö e.g., `architecture_review` profile should have `page_suffix: " - Architecture Review"` (or empty), not the developer suffix.
